# Contributing to NxtAbroad AI

Thank you for considering contributing to this project.

## 1. Fork & Branch
- Fork the repository
- Create a new branch:
  git checkout -b feature/my-feature

## 2. Code Standards
- Follow PEP 8 formatting
- Use clear, readable variable names
- Add docstrings to functions and classes

## 3. Testing
Before submitting a PR:
- Run the API locally
- Test `/predict` endpoint using sample JSON files

## 4. Commit Messages
Use descriptive commit messages:
- Add new rules
- Fix CSV parser
- Improve scoring logic

## 5. Pull Requests
All PRs must:
- Describe the purpose of the change
- Include tests or sample input/output
- Pass CI checks (once enabled)

## 6. Security
Do NOT commit:
- Real student or client data
- API keys or secrets
- Production environment files
